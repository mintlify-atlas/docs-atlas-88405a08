`pkg` supports two kinds of hooks: **lifecycle hook scripts** stored in `~/.config/pkg/scripts/` that run on every matching sync, and **per-package hooks** declared inline in your TOML group files that run only once at install time.

## Lifecycle hook scripts

Scripts placed in `~/.config/pkg/scripts/` are discovered automatically by name. You write and maintain them — `pkg init` creates editable examples to get you started.

### Naming convention

| Script name | When it runs |
|-------------|-------------|
| `before-sync.sh` | Before the entire sync starts |
| `after-sync.sh` | After the entire sync completes |
| `before-{backend}.sh` | Once before any packages for that backend are installed (e.g. `before-paru.sh`) |
| `after-{backend}.sh` | Once after all packages for that backend are done (e.g. `after-flatpak.sh`) |

The backend hook variants fire **once per sync per backend**, not once per group. If you have three groups that all use `paru`, `before-paru.sh` runs once before all of them and `after-paru.sh` runs once after all of them.

### Execution order during a full sync

```
before-sync.sh
  └── before-pacman.sh      ← if any group uses pacman
      [install pacman packages from all groups]
  └── after-pacman.sh
  └── before-paru.sh        ← if any group uses paru
      [install paru packages from all groups]
  └── after-paru.sh
  └── before-flatpak.sh     ← if any group uses flatpak
      [install flatpak packages from all groups]
  └── after-flatpak.sh
after-sync.sh
```

Scripts that do not exist are silently skipped — you only need to create the ones you want.

### Scripts must be executable

If a script exists but is not executable, `pkg` prints a warning and skips it:

```bash
# Make a hook executable
chmod +x ~/.config/pkg/scripts/before-sync.sh
```

You can check the status of all hook scripts with `pkg status`, which shows a `✓` or `✗ not executable` marker for each script found in the scripts directory.

### Environment variables

Every hook script receives these environment variables:

| Variable | Example value |
|----------|--------------|
| `PKG_EVENT` | `"before-sync"`, `"after-flatpak"` |
| `PKG_CONFIG_DIR` | `/home/user/.config/pkg` |
| `PKG_DATA_DIR` | `/home/user/.local/share/pkg` |

### Real examples from this dotfiles repo

The `before-sync.sh` script generated by `pkg init` — edit to uncomment the parts you want:

```bash
#!/usr/bin/env bash
# ~/.config/pkg/scripts/before-sync.sh
# Runs before EVERY `pkg sync`. Edit freely — runs each time, not just once.
#
# Available env vars:
#   PKG_EVENT       = "before-sync"
#   PKG_CONFIG_DIR  = path to ~/.config/pkg
#   PKG_DATA_DIR    = path to ~/.local/share/pkg
set -euo pipefail

echo "[pkg hook] before-sync — $(date '+%Y-%m-%d %H:%M')"

# Uncomment to pull dotfiles before installing new packages:
# cd ~/dotfiles && git pull --ff-only

# Uncomment to check for pending system updates first:
# checkupdates || true
```

The `after-sync.sh` script for re-applying dotfiles and sending a desktop notification:

```bash
#!/usr/bin/env bash
# ~/.config/pkg/scripts/after-sync.sh
# Runs after EVERY `pkg sync`. Edit freely — runs each time, not just once.
#
# Available env vars:
#   PKG_EVENT       = "after-sync"
#   PKG_CONFIG_DIR  = path to ~/.config/pkg
#   PKG_DATA_DIR    = path to ~/.local/share/pkg
set -euo pipefail

echo "[pkg hook] after-sync — $(date '+%Y-%m-%d %H:%M')"

# Uncomment to apply dotfiles via stow after all packages are in place:
# cd ~/dotfiles && stow --restow --ignore='.git' .

# Uncomment for a desktop notification when sync finishes:
# notify-send "pkg sync" "All packages installed" --icon=package-x-generic
```

<Tip>
  Hook scripts are always run, even if all packages are already installed. Put operations that should happen on every sync — pulling dotfiles, stowing configs, checking for updates — in a hook script, not in per-package hooks.
</Tip>

## The `--strict-hooks` flag

By default, a hook that exits non-zero prints a warning and `pkg sync` continues. Pass `--strict-hooks` to abort the entire sync on any hook failure:

```bash
pkg sync --strict-hooks
```

When a hook fails in strict mode, `pkg` exits with the hook's exit code. Any hooks that have not yet run are skipped. Already-installed packages remain installed.

Use `--strict-hooks` in automation or CI where a broken dotfiles state should be a hard error rather than a warning you might miss.

## Per-package hooks

Per-package hooks are declared inline inside your package group TOML files. Unlike lifecycle scripts, they run only once — at the moment the package is first installed. They do not run on subsequent syncs once the package is already present.

```toml
[paru]
packages = [
  "fastfetch",
  { name = "sddm", hooks = { after_install = [
    "sudo", "systemctl", "enable", "sddm"
  ] } },
  { name = "swayidle", hooks = { after_install = [
    "systemctl", "--user", "add-wants", "niri.service", "swayidle.service"
  ] } },
  { name = "dms-shell-bin", hooks = { after_install = [
    "systemctl", "--user", "add-wants", "niri.service", "dms"
  ] } },
  { name = "dsearch-bin", hooks = { after_install = [
    "systemctl", "--user", "enable", "--now", "dsearch.service"
  ] } },
]
```

### Available per-package hook keys

<ParamField path="before_install" type="list of strings">
  Command to run immediately before the package is installed. A non-zero exit is logged but does not stop the sync.
</ParamField>

<ParamField path="after_install" type="list of strings">
  Command to run immediately after the package is installed and recorded in the database. A non-zero exit is logged but does not stop the sync.
</ParamField>

### When per-package hooks fire

The sequence for a single package with both hooks:

```
before_install hook runs
  ↓
pkg installs the package
  ↓
package recorded in database
  ↓
after_install hook runs
```

If the package was already installed when `pkg sync` runs, neither hook fires.

<Note>
  Both per-package hooks are best-effort. A non-zero exit code is logged as a warning and the sync continues. There is no `--strict-hooks`-style flag for per-package hooks.
</Note>

## Summary: lifecycle scripts vs per-package hooks

| | Lifecycle scripts | Per-package hooks |
|--|--|--|
| Location | `scripts/*.sh` | Inline in `packages/*.toml` |
| Runs | Every matching sync | Only at install time |
| On failure (default) | Warning, continue | Warning, continue |
| On failure (`--strict-hooks`) | Abort sync | N/A |
| Use for | Dotfile sync, notifications, update checks | `systemctl enable`, one-time setup |
